# Monitoring and Observability

This directory is meant for documenting Service Level Agreements (SLAs), Service Level Objectives (SLOs), and Service Level Indicators (SLIs).

## Recommended Contents:

1. **SLAs/SLOs Definition**: Define what acceptable performance looks like for the business (e.g., "The API must have a 99.9% uptime per month" or "The P95 response time for the login route must be under 200ms").
2. **Dashboard Documentation**: Links and explanations of the most critical Grafana/Datadog dashboards.
3. **Alerting Rules (Runbooks)**: What the on-call developer should do when a specific alert (e.g., "High CPU on Database") is triggered.
