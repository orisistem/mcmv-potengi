# Database Architecture & Modeling

This folder is dedicated to the documentation of the database structure.
Before writing code or migrations, the database should be modeled here.

## Recommended Contents:

- **MER/DER (Entity-Relationship Diagrams)**: Visual representation of tables and relationships (can be image files or links to tools like dbdiagram.io or Draw.io).
- **Data Dictionary**: A document explaining the purpose of each table, descriptions of columns, their data types, and specific constraints.
